1. תמצית מנהלים
================================================================================

לפני התיקון: 51 דפים מאונדקסים, 172 "Crawled - currently not indexed" — בעיקר
בגלל article.html?slug= עם HTML ריק ("Loading article..."), סתירות sitemap/noindex,
ודפי ערך שלא ב-sitemap.

אחרי התיקון (מבני):

  • מאמרי אוטומציה מוגשים כ-Jekyll סטטי: /YYYY/MM/DD/slug/ עם canonical ב-front matter
  • Sitemap נקי מ-noindex ומ-URLים כפולים (query string הוסרו לאחר backfill)
  • Bing לא מואט ע"י Crawl-delay
  • Golden Triad (sitemap + llms.txt + footer) לדפי compare/glossary/MCP
  • תהליך עתידי מתועד ב-seo-geo-guardrails.md

עדיין פתוח:

  • אימות Google Search Console (שדה ריק עד להדבקת token)
  • Cannibalization בנושאי vibe coding (נפח תוכן גבוה)
  • זמן עד ש-Google יסרוק מחדש ויאנדקס (שבועות)
  • דף הבית / pricing — לא הורחבו במסגרת Pareto (how כן)

הערה: geo-context.js ו-/api/geo/context = גיאולוקציה למוצר, לא GEO למנועי AI.

2.1 ארכיטקטורת אתר (מעודכן)
---------------------------
  • Frontend: Jekyll (GitHub Pages) — דפים סטטיים + כל המאמרים ב-_posts/

  • article.html?slug= — legacy; 301 ב-_redirects ל-permalink סטטי
  • Backend (Render): sitemap, אוטומציית מאמרים, כתיבת _posts/ (מקומי או GitHub API)
  • מודול: backend/server/jekyll-post-writer.js

2.2 קבצי תשתית
----------------
  sitemap.xml          — 145 URLs (backend/server/sitemap-generator.js)
  robots.txt           — ללא Crawl-delay; Allow לבוטי AI
  _redirects           — redirects היסטוריים + בלוק מאמרי אוטומציה (~83)
  _config.yml          — permalink: /:year/:month/:day/:title/

2.3 מטא-תגים (_includes/head.html)
------------------------------------
  ללא שינוי מבני — כולל canonical, OG, Twitter, hreflang.
  google-site-verification: מוכן ב-Liquid; ערך ב-_config.yml עדיין ריק.

2.4 Schema.org
--------------
  ללא שינוי מבני. פוסטי אוטומציה ב-Jekyll מקבלים Article דרך layout: post.
  article.html — JSON-LD ב-JS רלוונטי בעיקר לגולשים שלא עברו 301.

2.5 סוגי תוכן לאינדוקס (מעודכן)
-------------------------------
  א. פוסטי Jekyll (_posts/) — ~127
     • כולל 44 פוסטים מקוריים + מאמרים מ-Firestore (backfill + אוטומציה עתידית)
     • URL: https://specifys-ai.com/YYYY/MM/DD/slug/
     • front matter: layout post, title (seo_title), description, canonical_url,
       tags, author, source: automation (למאמרים אוטומטיים)
     • גוף: content_markdown ללא H1 (ה-title הוא H1)

  ב. Firestore (articles) — מקור נתונים; לא URL ראשי לאינדוקס
     • אחרי backfill: jekyll_permalink על כל מאמר שפורסם

  ג. דפי שיווק — HTML סטטי; how.html הועשר (~300+ מילים ב-main)

2.6 robots.txt
--------------
  Disallow: auth, profile, spec-viewer, unsubscribe, admin dashboards, וכו'

  אין Crawl-delay (תוקן)
  Sitemap: https://specifys-ai.com/sitemap.xml

2.7 דפים עם noindex — לא ב-sitemap (תוקן)

8. מה עדיין מגביל שיפור בדירוג (ציפיות ריאליות)
================================================================================

8.1 לא ייפתר מיד
----------------
  • Google צריך לסרוק מחדש 145+ URLs ולהסיר זיכרון של article.html?slug=
  • 172 "Crawled not indexed" ירדו לאט — בקש indexing ידנית ל-10–20 URLs מרכזיים
  • Cannibalization בין 127+ מאמרים על vibe coding / AI tools
  • Authority / backlinks — מחוץ לטכני

8.2 תלוי בפריסה
---------------
  אם production חסר _posts/ או _redirects מה-commit — התיקון לא יופיע לבוטים.
  ודא deploy אחד הכולל: קוד + _posts/ + _redirects + sitemap.xml.

8.3 תפעול
----------
  search_console_verification, INDEXNOW_KEY, GITHUB_REPO על Render.
