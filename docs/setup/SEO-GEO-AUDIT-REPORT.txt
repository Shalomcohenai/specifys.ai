1. תמצית מנהלים
================================================================================

האתר כולל תשתית SEO ו-GEO יחסית מתקדמת (sitemap דינמי, llms.txt, דף hub
למנועי AI, schema.org, robots שמאפשר בוטי AI). עם זאת, יש פערים מבניים
שמונעים שיפור בדירוג ובציטוטים:

  • דפי השוואה (compare), MCP, Glossary — אינדקסבילים אך לא ב-sitemap
  • Academy category/guide ו-dynamic-post — ב-sitemap אך עם noindex (סתירה)
  • ~80 מאמרי Firebase — תוכן ומטא נטענים ב-JavaScript בלבד
  • נפח גבוה של תוכן אוטומטי על אותם נושאים (cannibalization)
  • אימות Google Search Console ריק ב-_config.yml
  • תוכן דק בדפי מוצר (מזוהה גם ב-docs/setup/seo.md)

הערה חשובה: הקבצים geo-context.js ו-/api/geo/context הם גיאולוקציה
(מדינה/locale למוצר), לא GEO למנועי בינה מלאכותית.

2. מבנה קיים — SEO
================================================================================

2.1 ארכיטקטורת אתר
------------------
  • Frontend: Jekyll (GitHub Pages) — דפים סטטיים, בלוג מ-_posts/
  • מאמרים דינמיים: Firebase + API — pages/article.html?slug=...
  • Backend: Node (Render) — יצירת sitemap, אוטומציית מאמרים, IndexNow
  • קובץ sitemap: שורש הפרויקט — sitemap.xml (מתעדכן אוטומטית)

2.2 קבצי תשתית
----------------
  sitemap.xml          — 140 URLs (נוצר ע"י backend/server/sitemap-generator.js)
  robots.txt           — Allow כללי + Disallow לאזורים פרטיים + Allow לבוטי AI
  _redirects           — מאות redirects מ-URLs ישנים
  _config.yml          — permalink בלוג: /:year/:month/:day/:title/
                         jekyll-sitemap מושבת — sitemap מהשרת

2.3 מטא-תגים גלובליים (_includes/head.html)
---------------------------------------------
  • title, description (מ-front matter של כל דף)
  • keywords — רשימה גלובלית קבועה לכל האתר
  • author: Specifys.ai
  • robots — ברירת מחדל: index, follow, max-image-preview:large, ...
              ניתן לדרוס per-page עם: robots: noindex, follow
  • canonical — מחושב מ-page.url או מ-page.canonical_url
  • hreflang: en + x-default
  • Open Graph: site_name, title, description, type, url, image (1200x630)
  • Twitter Card: summary_large_image, @specifysai
  • google-site-verification — ריק (לא מוגדר)
  • bing_webmaster_verification — ריק
  • Google Analytics: G-RWBT69JCM7

3. מבנה קיים — GEO (Generative Engine Optimization)
================================================================================

GEO = אופטימיזציה לנראות וציטוט במנועי תשובות AI (ChatGPT, Perplexity,
Claude, Google AI Overviews וכו') — מקביל ל-SEO עבור מנועי חיפוש קלאסיים.

3.1 נכסי GEO קיימים
-------------------
  קובץ              נתיב                    תפקיד
  ---------------------------------------------------------------------------
  llms.txt          /llms.txt               אינדקס markdown מסודר (מפרט llmstxt.org)
  ai.txt            /ai.txt                   מצביע קצר ultra-short ל-llms, hub, sitemap
  Hub HTML          /pages/for-ai-assistants  דף אנושי + machine-readable summary
  robots.txt        /robots.txt               Allow לבוטי AI
  sitemap.xml       /sitemap.xml              גילוי URLs + lastmod
  Glossary          /pages/glossary/          DefinedTerm schema — מונחים לציטוט
  Compare pages     /pages/compare/*          TL;DR + טבלאות — מענה לשאלות השוואה

3.2 תוכן llms.txt (תמצית)
-------------------------
  Entity: "AI app specification generator" (Specifys.ai)

  מקשר ל:
    • for-ai-assistants, ai.txt
    • דף הבית, how, why, pricing, ToolPicker, MCP, blog, academy
    • spec-viewer (מוזכר — אך noindex בפועל)
    • demo-spec, cursor-windsurf-integration, tools map
    • articles, about, contact

  לא מקשר ל (אך קיימים באתר):
    • דפי compare (×3) — רק ב-for-ai-assistants
    • glossary — רק ב-for-ai-assistants

3.3 דף for-ai-assistants.html
------------------------------
  • Entity summary (שם, תיאור, קהל יעד)
  • קישורים ל-llms.txt, ai.txt, sitemap, robots
  • רשימת "High-signal pages" כולל compare + glossary
  • FAQ עם JSON-LD FAQPage
  • WebPage + SoftwareApplication schema

3.4 דפי Compare (לא ב-sitemap, כן ב-footer/hub)
------------------------------------------------
  /pages/compare/specifys-vs-lovable.html
  /pages/compare/specifys-vs-bolt.html
  /pages/compare/specifys-vs-cursor-windsurf.html

  כל אחד: layout default, title/description ייעודיים, טבלת השוואה,
  TL;DR, JSON-LD — מכוון לתשובות "Specifys vs X".

3.5 Glossary
------------
  /pages/glossary/index.html
  מונחים: Vibe Coding, App Specification, MVP Brief, Mockup, Mind Map,
  Mermaid, Canonical URL, Schema Markup, ועוד.
  Schema: DefinedTermSet + DefinedTerm לכל מונח.

3.6 מה חסר ל-GEO
----------------
  [ ] קישור ל-llms.txt ב-<head> של האתר (rel alternate) — לא קיים
  [ ] סנכרון מלא בין llms.txt ↔ sitemap ↔ דפי compare/glossary
  [ ] תוכן מאמרים זמין ב-HTML ללא JS (חשוב ל-crawlers של AI)
  [ ] דף אינדקס מרכזי לכל דפי Compare
  [ ] מדידת ציטוטים (לא מיושם בקוד)

6. תגיות, Schema ו"וקטורים"
================================================================================

6.1 וקטור SEO (מסגרת הערכה)
----------------------------
  ממד                  מצב נוכחי
  ---------------------------------------------------------------------------
  Crawlability         טוב (robots, sitemap) — פערים בדפים חדשים
  Indexability         חלש — סתירות noindex/sitemap, JS articles
  On-page מוצר        חלש — thin content
  On-page בלוג/מאמרים  בינוני-חלש — נפח גבוה, אוטומציה
  Technical SEO        לא נבדק בדוח (Core Web Vitals)
  E-E-A-T / Backlinks  לא נבדק בקוד
  International        אנגלית בלבד — תקין

6.2 וקטור GEO
-------------
  ממד                        מצב
  ---------------------------------------------------------------------------
  Machine-readable index     טוב (llms.txt, ai.txt)
  Entity clarity             טוב (for-ai-assistants, SoftwareApplication)
  Citation-ready pages       בינוני — compare/glossary לא ב-sitemap/llms מלא
  AI crawler access          טוב (robots Allow)
  Content without JS         חלש — מאמרי Firebase
  Consistency llms/sitemap   חלש — פערים

6.3 בעיית מאמרים דינמיים (article.html)
-----------------------------------------
  HTML ראשוני לפני JS:
    title: "Article - Specifys.ai"
    body: "Loading article..."

  אחרי JS (updateSEOTags):
    title, description, canonical, OG, JSON-LD Article

  השלכה: Google מרנדר JS אך לא תמיד; בוטי AI פחות אמינים — חולשה ל-SEO ו-GEO.

8. למה הביצועים לא משתפרים
================================================================================

8.1 SEO אורגני (Google)
-----------------------
  1. Thin content בדפי מוצר — אין מספיק טקסט ייחודי לדירוג.
  2. Cannibalization — עשרות מאמרים על אותם keywords (Claude Code, Antigravity...).
  3. פער sitemap — דפי compare/MCP/glossary לא מגיעים דרך sitemap.
  4. סתירות — URLs עם noindex עדיין ב-sitemap.
  5. אין Search Console מאומת — אין נתונים לתיקון ממוקד.
  6. מאמרים ב-JS — אינדוקס איטי/לא שלם.
  7. תוכנית seo.md לא מומשה.

8.2 GEO (מנועי AI)
------------------
  1. יתרון יחסי — llms.txt + hub (רבים לא עושים).
  2. דפי compare/glossary — לא מסונכרנים ב-llms וב-sitemap.
  3. נפח מאמרים אוטומטיים — פחות authority ממקורות חזקים.
  4. תחרות entity — "app spec generator" מול Cursor, Lovable, Bolt.
  5. תוכן ראשוני "Loading..." — חלש ל-retrieval.

8.3 בלבול מונחים בפרויקט
-------------------------
  geo-context.js / api/geo/context = גיאוגרפיה למוצר (US/IL/DE recommendations)
  GEO במסמך זה = Generative Engine Optimization
