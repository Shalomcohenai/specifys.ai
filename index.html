Application Overview

Idea Summary

RelayDesk is a lightweight shared customer support inbox for indie SaaS founders. It unifies email threads, macros, and light SLA timers so a tiny team can answer customers without enterprise helpdesk complexity or seat-tax pricing.

Indie SaaS teams drown in a mix of personal Gmail, Notion docs, and Slack forwards. Important tickets get lost, replies collide, and founders cannot see response time. RelayDesk closes the gap with a focused shared inbox, reusable macros, and simple SLA lite timers.

Personas, epics, acceptance criteria, and screen states are part of the enriched Overview — the same shape Spec Viewer renders after generation.

Core Features

  • Shared inbox: team mailbox with assignable threads
  • Macros: reusable reply snippets with variables
  • SLA lite: first-response timers and breach badges
  • Email sync: IMAP/SMTP or provider OAuth for a support address
  • Thread status: open, pending, resolved — plus full-text search

Technical Specification

Architecture Overview

Browser SPA talks to RelayDesk API. API owns threads, macros, and SLA state in Postgres. An email worker syncs IMAP/OAuth mail and enqueues inbound messages. Object storage holds attachments. Stack: React 18 + Vite + TypeScript, Node.js 20 + Express, PostgreSQL 16, S3-compatible storage, email magic-link auth.

Database Schema

PostgreSQL relational model for workspaces, users, mailboxes, threads, messages, and macros — with an ER Mermaid diagram embedded in the enriched Technical stage (same shape Spec Viewer validates).

API Endpoints

  • GET /api/threads — List threads for workspace with filters
  • POST /api/threads/{id}/replies — Send a reply on a thread
  • POST /api/macros — Create macro
  • GET /api/macros — List macros for workspace
  • PATCH /api/threads/{id} — Update status or assignee
